随机森林是一种**集成算法**。通过在数据上构建多个模型，集成所有模型的建模结果。

集成算法的目标是：考虑**多个评估器的建模结果**，汇总后得到一个综合的结果，以此来获取比单个模型更好的回归或分类表现。主要有三种集成算法：
- **装袋法** bagging：核心是构建多个独立的评估器，互相平行；代表的模型是**随机森林**
- **提升法** boosting：每次建立模型时提高难以评估的样本的权重，构成一个强的评估器；模型是有序的
- stacking

多个模型集成的模型成为**集成评估器**，组成集成评估器的每个模型称为**基评估器**，在随机森林中，基评估器为决策树

`sklearn` 中集成算法集中在模块 `ensemble` 中，其中很大一部分都是和随机森林有关的。

## RandomForestClassifier
随机森林是具有代表性的 bagging 集成算法，基评估器为决策树，分类树组成的森林为随机森林分类器；回归树集成的森林称为随机森林回归器。

其中有的参数和分类的决策树一致，如：
- `criterion`
- `max_depth`
- `min_samples_leaf`
- `min_samples_split`
- `max_features`
- `min_impurity_decrease`

单个决策树的准确率越高，整体随机森林的准确率也越高。

### 重要参数
#### n_estimators
表示森林中的树木数量（即*基评估器的数量*）一般来说，越大越好，但是会有决策边界，达到一定的 `n_estimator` 后，精确率会不再上升而是发生波动。


#### random_state
随机森林通过**袋装集成算法**，对*基评估器的预测结果进行平均或多数表决*来确定集成评估器的结果。因此，在有 25 颗树的随机森林中，只有超过 13 颗树判断错误时，随机森林才会判断错误。

随机森林中，可以通过 `radnom_state` 来控制随机数，可以生成一组互不相同的树（源于*随机挑选特征进行分支*带来的随机性），随机性越大时，袋装法的效果越好，**袋装法集成时，基分类器是相互独立的**。

但是只有该随机性是有局限性的。当需求的树非常多时，而数据集有可能没有那么多的特征来生成互不相同的树。

#### bootstrap & oob_score
为了让基分类器都尽量不一样，袋装法用**有放回随机抽样技术**形成不同的训练数据。

可以设定采集 $m$ 个自助集，每个自助集中有 $n$ 个样本。

每个样本被抽到的概率为
$$
1 - \left( 1 - \frac{1}{n} \right)^{n} \approx \frac{1}{e}
$$
约为 36.8%。因此数据集中大约有 37% 的样本会被浪费，这些数据被称为**袋外数据**（out of bag data, oob）可以用作测试集。但是，当 $n$ 和 $n\_estimators$ 不够大时，可能没有袋外数据，那么将无法使用带袋外数据来测试。 

如果想用袋外数据来测试，需要传入 `oob_score=True` ，在训练后通过 `rfc.oob_score` 来查看在袋外数据上的测试结果。
```py
rfc_1 = RandomForestClassifier(n_estimators=25, oob_score=True)
rfc_1 = rfc_1.fit(Xtrain, Ytrain)
# 袋外数据的测试结果
rfc_1.oob_score_
```

### 重要属性和接口
和决策树的属性和接口一样，随机森林也有相关的属性：
- `estimators_`, `apply`, `score`, `fit`, `predict` 等

此外还有一个接口 `predict_proba`，返回每个测试样本被分到每隔一个标签的概率；传统的随机森林利用袋装法的规则，*平均或少数服从多数*来决定**集成**的结果；sklearn 中的随机森林通过返回每个样本对应的所属标签**概率均值**来决定样本的分类。


### 使用装袋法的一个必要条件
当一颗决策树的准确率可以确定时，整个随机森林产生错误判断的概率可以通过排列组合来计算，
$$
e_{random\_forest} = \sum_{i= \lceil n / 2 \rceil }^{n} C_{i}^{n} \epsilon^{i}(1- \epsilon)^{n - i}
$$
因此，当一棵树的判断错误的可能性 $\epsilon$ 超过一定值时，袋装的集成算法就失效了，随机森林就不如**基分类器**的准确了。

### 随机森林填补缺失值
`sklearn.impute.SimpleImputer` 可以将均值/中值等填补缺失值
 
用随机森林填充缺失值的思想：
特征矩阵和标签之间存在联系，而实际上标签和特征是可以互换的，可以把标签放到特征矩阵中，再从特征矩阵中选取一个标签作为缺失值的填充对象；*剩下的标签和特征一起可以预测这个特征*。

对于一个有 n 个特征的数据，其中的特征 T 有缺失值，可以将其他的 $n-1$ 个特征和*原本的标签*组成新的特征矩阵，而*特征 T 中没有缺失的部分*就是 `Ytest`
- 特征 T **不缺失的值**对应的其他 $n-1$ 个特征+原本的标签：`Xtrain`
- 特征 T **不缺失的值**：`Ytrain`
- 特征 T **缺失的值**对应的其他 $n-1$ 个标签+原本的标签：`Xtest`
- 特征 T 缺失的值：未知，需要预测。

如果有多个特征都有缺失，填充的方法是从**缺失少的特征**开始填充。填补一个特征时，先将其他特征的缺失值**用 0 代替**，每完成一次回归预测，将结果放入特征矩阵中，再填充下一个特征。
- 找出数据集中，缺失值从小到大排列的特征顺序，用 `np.argsort`


```py
sorted_index = np.argsort(X_missing.isnull().sum()).values

X_missing_reg = X_missing.copy()

print(X_missing_reg.shape)
for i in sorted_index:
    print(i)
    # 每个i是需要填充缺失值的列
    
    # 需要取出非填充列和标签列组成特征矩阵
    # 填充列作为 Y
    
    # 需要填充的列
    fillc = X_missing_reg.iloc[:, i]
    print(fillc.info())
    
    # 构造新的特征矩阵
    df = pd.concat([pd.DataFrame(X_missing_reg.iloc[:, X_missing_reg.columns != i]), pd.DataFrame(Y_missing)], axis=1)
    
    # 特征矩阵中的缺失值用0填充
    df = SimpleImputer(missing_values=np.nan, strategy="constant", fill_value=0).fit_transform(df)
    
    # 构造训练和测试集
    Ytrain = fillc[fillc.notnull()]
    Ytest = fillc[fillc.isnull()]
    
    Xtrain = df[fillc.notnull()]
    Xtest = df[fillc.isnull()]
    print(Xtrain.shape, Xtest.shape)
    
    rfc = RandomForestRegressor(n_estimators=25, random_state=42)
    rfc.fit(Xtrain, Ytrain)
    Ypredict = rfc.predict(Xtest)
    
    X_missing_reg.iloc[X_missing_reg.iloc[:,i].isnull(), i] = Ypredict
    

X_missing_reg.isnull().sum()
```



## 机器学习中调参的基本思想
模型调参的第一个目标为找准目标。对于随机森林来说，需要提升的是模型在未知数据上的准确率，即**泛化误差**。
当模型过于复杂和过于简单时泛化误差都会很大，
- 模型复杂度高时，对数据学习太过具体，导致*过拟合*
- 当模型太简单时，模型的学习不够，误差也会大，会发生*欠拟合*

对于树模型来说，树越茂盛，深度越深，枝叶越多，模型就越复杂。因此，随机森林天生是一种**复杂度高的模型**。因此调参的目标是减少模型的复杂度；但也不是绝对的，有的数据决策树天生难以学习。总之，调参前需要判断模型所处泛化误差曲线的位置。总结有一下的结论：
- 模型太复杂或者太简单都会让泛化误差增加，追求的是中间的平衡点
- 模型太复杂会导致过拟合；太简单会导致欠拟合
- 对树的模型来说，树的深度越深，枝叶越多，模型越复杂
- 树模型和树集成的模型的调参目标都是减少模型的复杂度

一般来说，树模型的参数重要程度关系由如下表所示：

| 参数                 | 影响程度                                                           |
| ------------------ | -------------------------------------------------------------- |
| `n_estimators`     | 单调；一般越大表现越好；但是不影响模型的复杂度                                        |
| `max_depth`        | 默认是最大深度；减少参数可以让模型更简单                                           |
| `min_samples_leaf` | 默认是最小限制，即最高复杂度。让参数增加，向着模型复杂度更低的方向调参。同样的参数有 `min_samples_split` |
| `max_features`     | 默认是 auto，为特征总数的平方根，出于中间复杂度；对模型的影响不大；减小时可以让模型更简单；增大时可以让模型更复杂    |
| `criterion`        | 一般为 gini，效果有增有减                                                |
