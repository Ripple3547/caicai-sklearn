## 数据挖掘
1. 数据挖掘
2. 数据预处理：从数据中*检测*/*纠正*或删除损坏或不准确和不适用于模型的记录；让数据更适应模型，匹配模型需求
3. 特征工程：将原始数据转换为*更能代表预测模型的潜在问题的特征*的过程，通过提取相关的特征或者创造特征实现；可能面对的问题有：有*无关的特征*；特征之间有*相关性*；特征太多或太小；目的是：降低**计算成本**，提升**模型上限**

##  数据预处理
### 数据无量纲化
将不同规格的数据转换到同一规格，转换到某个特定的分布。可以提升模型的精度，防止防止取值范围大的特征对模型产生影响。但是决策树不需要**无量纲化**，可以处理任何的数据。

包括线性和非线性的处理方法
- 线性的无量纲化包括**中心化**和**缩放处理**


#### 数据归一化
将数据按照最小值中心化，再按极差缩放，Normalization（不是正则化）
$$
x^{*} = \frac{x-\min (x)}{\max (x) - \min (x)}
$$
归一化后的数据服从**正态分布**。

模块 `sklearn.preprocessing.MinMaxScaler`
实例化对象后可以通过 `fit` 以及 `transform` 将数据归一化；如果数据量比较大，则可以使用 `partial_fit` 归一化

```py
scaler = MinMaxScaler()
scaler.fit(X)

X_scaled = scaler.transform(X)

# 还愿
X_returned = scaler.inverse_transform(X_scaled)
X_returned
```

或使用 `numpy`
```py
X_array = np.array(X)
X_scaled_numpy = (X_array - X_array.min(axis=0)) / (X_array.max(axis=0) - X_array.min(axis=0))

X_scaled_numpy * (X_array.max(axis=0) - X_array.min(axis=0))  + X_array.min(axis=0)
```

#### 数据标准化
数据按均值中心化，再按标准差缩放，变成均值为 0，方差为 1 的正态分布。

```py
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
scaler.fit(X)

scaler.mean_, scaler.var_

X_standard = scaler.transform(X)
X_standard
```

在标准化和归一化时，**空值**（NaN）在 `fit` 时会被直接忽略，而在 `transform` 后也会保持 NaN 的形式；只允许导入*二维的数组*。

一般来说选用的是标准化来进行特征缩放。`MinMaxScaler` 对异常值敏感，过大的异常值会使得其他相对较小的数据被压缩的很小，因此一般用 `StandardScaler`。而 `MinMaxScaler` 对于不涉及距离度量/梯度/协方差计算或者数据需要被压缩到特定区间时适用。

中心化只需要 pandas 广播减去某个数即可。其他 `sklearn` 中的缩放处理模块有：
- `MaxAbsScaler` 希望数据压缩，同时不损失稀疏性（不影响取值为 0 的个数）
- `RobustScaler` 在数据噪声和异常值比较大时可用

### 缺失值
使用 `sklearn.impute.SimpleImputer` 填充缺失值，包括 4 个重要的参数
- `missing_values` 缺失值时什么，默认是 `np.nan`
- `strategy` 填补缺失值的策略，有 `mean`，`median`，`most_frequent`，`constant`（需要配合 `fill_value`），*默认* `mean`
- `fill_value` 当参数 `strategy` 为 `constant` 时可用，表示填充的值，*默认 0*
- `copy` 默认为 True，表示创建一个副本

```py
from sklearn.impute import SimpleImputer

# 实例化类
imp_mean = SimpleImputer()
imp_median = SimpleImputer(strategy="median")
imp_0 = SimpleImputer(strategy="constant", fill_value=0)

# 先填补age列
Age = data.loc[:, "Age"].values.reshape(-1,1)
Age.shape

imp_mean = imp_mean.fit_transform(Age)
imp_median = imp_median.fit_transform(Age)
imp_0 = imp_0.fit_transform(Age)

# 选择用中位数填补
data.loc[:, "Age"] = imp_median
data.info()
```

对于 embarked 特征，为字符串型分类变量，用众数填补
```py
embarked = SimpleImputer(strategy="most_frequent")
embarked.fit_transform(data.loc[:, "Embarked"].values.reshape(-1,1))

data.loc[:, "Embarked"] = embarked
```

当然，也可以用 pandas 填补
```py
data_.loc[:, "Age"] = data_.loc[:, "Age"].fillna(data_.loc[:, "Age"].median())
```


### 分类型特征编码
#### LabelEncoder
处理标签，将分类转换为分类数值，模块为 `sklearn.preprocessing.LabelEncoder`

```py
# 标签的编码
y = X_copy.iloc[:, -1]

le = LabelEncoder()
le.fit(y)
```

在 fit 过后，可以查看标签中具体有多少的类别：
```py
le.classes_
```

当然也可以通过 `le.inverse_transform(label)` 逆转编码后的标签。

之后将标签转为编码后的值，当然也可以通过 `fit_transform` 来整合整个过程。
```py
label = le.transform(y)
# 取的是 sex 和 embarked 列
X_copy.iloc[:, -1] = label
X_copy.head()
```

#### OrdinalEncoder
对于特征，可以通过 `sklearn.preprocessing.OrdinalEncoder` 来进行编码。

```py
from sklearn.preprocessing import OrdinalEncoder

OrdinalEncoder().fit(X_copy_ordinal.iloc[:, 1:-1]).categories_
```
取出需要编码的特征列，在 fit 后可以通过 `.categories_` 查看每个特征具体有多少的类

最终通过 `transform` 实现特征列的编码：
```py
X_copy_ordinal.iloc[:, 1:-1] = OrdinalEncoder().fit_transform(X_copy_ordinal.iloc[:, 1:-1])
```

可以在结果中发现每个特征都编码为了**数字类型**，但是这种编码无法体现变量之间的联系。
一些常见的分类变量有：
- *名义变量*，如 Embarked 的取值 S Q C,彼此之间无联系
- *有序变量*，如学历的关系，但是不能加和和计算
- *有距变量*：如体重的划分分类：>45, >90, >135，*可以通过运算相互转换*

上述的变量编码后会变成一串有序的数字，模型会觉得某个特征中的变量通过运算可以得到一种等价关系，对于 Embarked 和学历特征来说会引入一些错误信息。

因此 `OrdinalEncoder` 适用于一些有序变量，对于名义变量，只能使用**哑变量**来处理。

#### OneHotEncoder
通过独热编码，将特征转化为哑变量，模块 `sklearn.preprocessing.OneHotEncoder`

先取出需要编码的特征列
```py
x = X_copy.iloc[:, 1:-1]
```
然后实例化对象，得到编码结果：
```py
enc = OneHotEncoder(categories="auto").fit(x)
result = enc.transform(x)

result.toarray()
```
转为array 后，可以发现将特征列OneHot编码后得到一个稀疏矩阵，一共5列，表示总的特征类别之和。

同理可以逆转编码
```py
pd.DataFrame(enc.inverse_transform(result))
```

当总的特征分类数比较多时，哑变量可能无法直观看出属于哪一个特征分类变量，可以通过：
```py
enc.get_feature_names_out()
```
查看哑变量的**每一个维度对应的特征取值**。（高版本）


最终，把哑变量的结果放入特征矩阵中，并且去掉原来的特征列，再重命名列，完成哑变量编码
```py
X_copy_onehot = pd.concat([X_copy, pd.DataFrame(result.toarray())], axis=1)
X_copy_onehot.drop(["Sex", "Embarked"], axis=1, inplace=True)

newX = X_copy_onehot.copy()
newX.columns = ["Age", "Survived", "Female", "Male", "C", "Q", "S"]
```

也可以把上面的 feature names out 的结果作为列名重命名。

标签是否可以做哑变量（可以 `LabelBinarizer`）？因为用的是分类算法（如决策树），因此没有必要对标签做哑变量编码。


### 连续性特征编码
#### 二值化
模块 `sklearn.preprocessing.Binarizer`
年龄变量是**连续性变量**，可以设定一个阈值，将年龄二值化。二值化是对文本计数数据的操作。

如：对年龄和存活情况进行二值化
```py
from sklearn.preprocessing import Binarizer

x = data_2.iloc[:, 0].values.reshape(-1,1)
transformer = Binarizer(threshold=30).fit_transform(x)
```
关于边界值的处理策略：
> Values greater than the threshold map to 1, while values less than or equal to the threshold map to 0. With the default threshold of 0, only positive values map to 1.

大于阈值的会被编码为 1.

#### 分箱
模块 `sklearn.preprocessing.KBinsDiscretizer`
将连续性变量划分为分类变量的类，

将年龄分为多个区间，分箱后编码，有如下的参数：
- `n_bins` **分箱的个数**（每个特征；一次会被运用到所有导入的特征，如果想每个特征的分箱个数不同，则需要一个个导入）
- `encode` **编码**的方式：默认是 `onehot` 返回一个稀疏矩阵，每一列的一个特征中的一个类别。
	- `ordinal` 每个特征的每个箱编码为一个整数，返回的矩阵中每一列是一个特征，每个特征下含不同整数编码的箱。
- `strategy` 定义**箱宽**的方式，默认是 `quantile`
	- `uniform`，表示*等宽分箱*。即每个箱的宽度为 `极差/n_bins`，每个箱的样本数是不均匀的。
	- `quantile` 表示*等位分箱*，特征箱内的样本数量相同。
	- `kmeans` 表示按聚类分箱。


```py
from sklearn.preprocessing import KBinsDiscretizer

X = data_2.iloc[:, 0].values.reshape(-1,1)
est = KBinsDiscretizer(n_bins=3, encode="ordinal", strategy="uniform")
est.fit(X)
est.transform(X)
pd.DataFrame(est.transform(X))[0].unique()

# one hot 分箱
est = KBinsDiscretizer(n_bins=3, encode="onehot", strategy="uniform")
est.fit(X)
est.transform(X).toarray()
```

## 特征选择
特征工程的内容：
- **特征提取**：从图像等非结构化数据中提取新的信息
- **特征创造**：将数据中的原有特征进行计算，得到一个新的特征。
- **特征选择**：从所有的特征中选择对模型有意义的。

做特征选择的第一步是理解业务。

```py
import pandas as pd
data = pd.read_csv("digit recognizor.csv")

X = data.iloc[:, 1:]
Y = data.iloc[:, 0]

X.shape
```
可以发现这个数据集的维度十分大，有 784 维，一般的算法难以处理，因此需要特征工程。

### Filter 过滤法
过滤法用作预处理的步骤。特征选择独立于机器学习算法。过滤法是根据各种统计指标来选择特征的。从所有的特征中选择最佳的特征子集。

#### 方差过滤
##### `VarianceThreshold`
通过特征本身的方差来筛选特征。
- 如果一个特征本身的方差很小，表明在这个特征中大多数的数值都一样。

因此要优先消除**方差为 0** 的特征。
```py
from sklearn.feature_selection import VarianceThreshold

selector = VarianceThreshold()
X_var0 = selector.fit_transform(X)

X_var0.shape
```
可以发现过滤前后的特征减少了 76 个。

查看过滤的特征：
```py
# 输入的特征
selector.feature_names_in_

# 输出的特征
selector.get_feature_names_out()

# 过滤的特征
selected = set(selector.feature_names_in_) - set(selector.get_feature_names_out())
```

此外可以根据方差的中位数过滤特征：
```py
x_fsvar = VarianceThreshold(np.median(X.var().values)).fit_transform(X)
```
若特征服从伯努利分布，则可以过滤某个分类中占到 80%以上的特征：
```py
x_bvar = VarianceThreshold(.8 * (1 - .8)).fit_transform(X)
```

接下来测试 KNN 算法和随机森林对于方差过滤后的运算效率：
```py
from sklearn.ensemble import RandomForestClassifier as RFC
from sklearn.neighbors import KNeighborsClassifier as KNN
from sklearn.model_selection import cross_val_score
import numpy as np

cross_val_score(KNN(), X, Y, cv=5).mean()
# 方差过滤后
cross_val_score(KNN(), x_fsvar, Y, cv=5).mean()

cross_val_score(RFC(n_estimators=10, random_state=42), X, Y, cv=5).mean()

cross_val_score(RFC(n_estimators=10, random_state=42), x_fsvar, Y, cv=5).mean()
```
发现：
- 随机森林的准确率（93.9%）不如 KNN 算法（96.6%）
- 但是随机森林的运行时间（16.3s）更少（27.6s）
- 发现方差过滤后 KNN 的运行时间变化很大（减小了约 6 s），而随机森林的变化很小（几乎没有变化，甚至可能增加）。（随机森林只选择若干个特征；而 KNN 需要遍历所有的特征，时间复杂度高）

因此，**过滤法**的主要对象是需要*遍历特征*或者*升维*的算法，主要目的是*维持算法表现的前提下，降低算法的计算成本*。
