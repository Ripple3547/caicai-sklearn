## 概述

决策树是一种非参数的有监督学习方法
- **非参数**：输入的数据可以是任何类型的
- **有监督**：学习需要有答案

从数据中总结出决策规则，用一棵来表示，树中的每一个分支表示了对一个标签的分类，最终叶子节点就是要判断和输出的标签。

如对于下面的数据
![[image.webp]]

将所有的动物分类，画出下面的决策树
![[image-1.webp|263]]
对**特征进行提问**，根据不同的走向来确定某个动物是否是哺乳动物。比如，鲑鱼是冷血的，根据决策树可以直接判断是*非哺乳动物*。


下面是决策过程中的节点类型：
- 最初问题所在的节点是*根节点*
- 得到结论前的每一个问题都是*中间节点*，出边可以**有很多条**
- 而每一个结论都是*叶子结点*

---
### 决策树算法的核心解决问题

- 如何从数据集中找出一个最佳的节点划分和分枝
	- 如何确定要**向哪一个特征提问**；如何确定提问之后的下一个问题？*如何找出最佳节点和最佳分支*？
- 如果数据集有很多个特征，那么是否要对**所有的特征**都进行提问？如何**高效分类**？即如何*让决策树停止生长，防止过拟合*

使用的基本流程为：
1. **实例化**：调用需要使用的类，建立模型
2. **训练**：调用相关的接口训练模型，一般用 `fit`
3. **获取信息**：通过相关接口获取信息 `score`

如：
```py
from sklearn import tree

clf = tree.DecisionTreeClassifier()
clf = clf.fit(x_train, y_train)
result = clf.score(x_test, y_test)
```


## `DecisionTreeClassifier` 
分类树的相关接口为 `sklearn.DecisionTreeClassifier(...)`

有若干个重要的参数：


### 重要的参数
#### Criterion
用于决策树确定找到最佳节点分类的底层算法
- **entrpy**表示使用*信息熵*计算。计算基于信息熵的信息增益（父节点和子节点的**信息熵的差额**为信息增益）
- **gini**表示使用*基尼系数*计算


衡量一个决策树是否最佳的一个重要指标是*不纯度*（**Impurity**），每个节点都有一个不纯度（*越低越好*），相关性质为：
- **子节点**的不纯度一定低于**父节点**
- **叶节点**的不纯度最低。

上述两种计算方法的计算公式分别为
$$
\begin{align}
Entropy(t)  & = - \sum_{i=0}^{c-1} p(i | t) \log_{2} p(i|t) \\
Gini(t)  &  = 1 - \sum_{i = 0}^{c-1} p(i|t)^{2}
\end{align}

$$
两者的训练效果相当，但是信息熵对不纯度更加敏感，惩罚更强。但由于其对不纯度的敏感性，对训练集的拟合会更好，对于*高维*或*噪声比较大*的数据集容易**过拟合**。


默认的参数为*gini*系数；当欠拟合时，可以用*entropy*

---
决策树的基本计算流程为
1. 计算*全部指标*的不纯度
2. 选取其中*不纯度最优的指标*进行分支
3. 再次计算不纯度最优的指标，进行分支
4. 直到**没有更多的特征**可用；或当前的**不纯度最优**

```py
# 数据集的划分
from sklearn.model_selection import train_test_split

Xtrain, Xtest, Ytrain, Ytest = train_test_split(wine.data, wine.target, test_size=0.3)
```

---

#### random_state  & splitter


```py
clf = tree.DecisionTreeClassifier(criterion="entropy")
clf.fit(Xtrain, Ytrain)
clf.score(Xtest, Ytest)
# 可以发现每次运行的结果不同
```
每次会**从所有的特征中选取若干个特征**，并构造一棵决策树，在所有的决策树中找到一颗最好的树，因此最终的训练结果是随机的。
- 当数据集比较大时，随机性带来的结果波动会比较大
- 而对于比较小的数据集（如 iris），特征比较少，每次运行的结果相对固定
- 可以指定一个 `random_state=42` 固定随机种。


`splitter` 的参数两种选择：
- **best**表示优先选择重要的特征进行分枝（`clf.feature_importances_`）
- **random**表示随机选择，可以*防止过拟合*。同时可以发现生成的树规模变大了

---

#### 剪枝参数 1
- `max_depth`：超过限定深度的分支不再产生，即限制了整棵树的深度
- `min_samples_leaf`：一个节点**分出的子节点**的样本数至少为该值 
	- 如果选的过小，会引起过拟合
	- 如果选的过大，无法让模型学习数据
	- 输入一个浮点数可以根据样本量的百分比来剪枝
	- 一般选择 **5** 最合适，并配合 `max_depth` 使用
	- 类别不多时，**1** 通常最佳
- `min_samples_split`： 一个节点的样本量至少为该值才能进行**分支**。

![[image-3.webp|454]] 
加上原先的 score，进行不同的剪枝参数变化后的 score 变化为

| 策略                     | 得分    |
| ---------------------- | ----- |
| **无剪枝**                | 0.833 |
| `max_depth=3`          | 0.926 |
| `min_samples_leaf=3`   | 0.889 |
| `min_samples_split=13` | 0.778 |

在此次的训练中，可以发现适当限制深度确实可以增强拟合效果。


---
#### 剪枝参数 2
- `max_features`：限制最多可以使用的特征个数，如在 wine 数据集中一共有 13 个特征，如果设定 `max_features=6`，那么决策树中最多会存在 6 个特征；可以限制*高维度下的过拟合*，方式比较暴力，没有考虑到**各个特征的重要程度**，容易导致学习不足。如果想通过降维来防止过拟合，可以用 **PCA **。
- `min_impurity_decrease` 限制信息增益下降大小（*父节点*和*子节点*之间的**信息增益的差值**）；如果信息增益越大，说明这一层特征的选择效果越大。限制最小的信息增益，如果父子节点之间的信息增益差距过小，则不发生分支。（added since *sklearn 0.19*）

为了方便调参，我们可以画**超参数曲线**：根据不同的取值，绘制相关的得分曲线：
```py
test = []

for i in range(10):
    clf = tree.DecisionTreeClassifier(max_depth=i+1, random_state=42, splitter="random", criterion="entropy")
    clf.fit(Xtrain, Ytrain)
    score = clf.score(Xtest, Ytest)
    test.append(score)
    
plt.plot(range(1,11), test)
plt.xlabel("Max Depth")
plt.ylabel("Test Accuracy")
plt.title("Decision Tree Performance")
plt.show()
```

#### 目标权重参数
- `class_weight`：某标签的一类可能占据了很大的比例，如否的占比远大于是的比例，那么直接回答*否*也能有很大的准确率。因此我们期望让少的标签获得更大的权重，让模型跟偏向少数类。参数默认为**None**，表示会自动计算
- `min_weight_fraction_leaf`：有了权重，样本量的计算会受到权重的影响，需要搭配此参数（基于权重的剪枝参数），相比不知道样本权重的标准（`min_samples_leaf`）更**少偏向主导的类**。


### 重要的属性和接口
决策树类还有两个重要的接口
- `apply`：返回每个测试样本在叶子节点的索引
- `predict`：测试样本的分类/回归结果

这两个参数都只需要测试的 `Xtest` 数据即可。不接受一维特征的矩阵输入，需要通过 `reshape(...)` 来实现增维。


注意，决策树对应**环形分割的数据集**的分类效果不佳，在划分数据集和测试集时，需要注意防止数据集出现环形。
- 一个直观的理解是每次决策树分类时，会画出一个超平面（**决策边界**），随着分类的增加不断完善，但是对于环形数据难以处理。



## 回归树 DecisionTreeRegressor
回归树中，没有标签分布是否均衡的问题，因此没有 `class_weight` 的参数。


### 重要参数/属性/接口
#### Criterion
有三种指标：
- `mse`：均方误差：根据*父节点*和*子节点*的 mse 的差额做特征选择；通过叶节点的均值来最小化 $\mathscr{l}_{2}$ Loss。在回归树中，MSE 不仅时**分支质量的衡量指标**，同时也是**回归质量衡量指标**。
- `friedman_mse`：费尔德曼均方误差，为改进后的均方误差。
- `mae`：绝对平均误差，用叶节点的中值最小化 $\mathscr{l}_{1}$ Loss。

而 score *默认*返回的不是 MSE，而是 $R^{2}$，定义如下：
$$
R^{2} = 1 - \frac{u}{v}
$$
其中：
- $u = \sum_{i=1}^{n}(f_{i} - y_{i})^{2}$ 残差平方和
- $v = \sum_{i=1}^{n}(y_{i} - \hat{y})^{2}$ 总平方和

*可正可负，最大为 1*.

**均方误差一定为正**，但在 sklearn 中计算返回的是**负的均方误差**。原因在于均方误差被视作为一种误差，为loss，因此都用负数表示。

### 交叉验证
类似决策树的生成和筛选策略，交叉验证将数据划分为 n 份，每次选其中 1 份为测试集（每次选的不同）；$n-1$ 份作为训练集，重复 $n$ 次，将 $n$ 次结果取平均。是观察模型稳定性的一种方法。
```py
from sklearn.datasets import fetch_california_housing
from sklearn.model_selection import cross_val_score
from sklearn.tree import DecisionTreeRegressor

housing = fetch_california_housing()
regressor = DecisionTreeRegressor(random_state=42)
cross_val_score(
    regressor, 
    housing.data, 
    housing.target, 
    cv=10,
    # 不填scoring 默认返回R^2
    scoring="neg_mean_squared_error"
    )
```
- 第一个参数是任何的机器学习模型
- 第二个参数：完整的特征矩阵，*不需要划分测试集和训练集*。
- 第三个参数：target 标签
- 第四个参数 `cv` 表示做交叉验证的次数（即数据集*被分的份数*）
- 第五个参数 `scoring="neg_mean_squared_error"` 如果不填，默认为 $R^{2}$。
